<objective>
Make failed background auto-compaction a first-class terminal outcome that core can consume exactly once.

Purpose: Phase 3 made successful background compactions durable, but failed outcomes are currently stored and then cleared without recovery. Phase 4 starts by tightening the active-turn state machine so failed outcomes survive long enough to drive deterministic recovery without disturbing cancellation semantics.
Output: core turn state and teardown behavior support exactly one terminal outcome per background compaction: applied, failed-and-ready-for-fallback, or aborted.
</objective>
